# Практична робота "Поглиблене використання масивів"

## Завдання: Сортування матриці

Цей проект реалізує **Завдання 8** з практичної роботи:

### Описання задачі
Необхідно відсортувати квадратну матрицю заданого порядку N в порядку зростання елементів (зліва-направо, зверху-вниз).

#### Принцип роботи:
- Матриця розглядається як послідовність елементів від верхнього лівого кута до нижнього правого
- Елементи витягуються у одновимірний масив у порядку: перший рядок, другий рядок і т.д.
- Одновимірний масив сортується у порядку зростання за допомогою `Arrays.sort()`
- Відсортовані елементи повертаються у матрицю у тому ж порядку (зліва-направо, зверху-вниз)

#### Приклад:
```
Вихідна матриця:          Відсортована матриця:
[9, 2, 5]                 [1, 2, 3]
[4, 7, 1]        ====>    [4, 5, 6]
[8, 3, 6]                 [7, 8, 9]
```

## Реалізація

### Структура проекту:
```
src/
├── MatrixSorter.java  - основний клас з логікою сортування
└── Main.java          - клас для тестування функціональності
```

### Ключові особливості реалізації:
✅ **Універсальність** - клас `MatrixSorter` не містить інтерфейсу користувача, тільки чисту логіку  
✅ **Використання Arrays** - використовується клас `java.util.Arrays` для сортування  
✅ **Javadoc коментарі** - повна документація методів і класу з поясненнями  
✅ **Валідація даних** - перевірка на порожню матрицю та квадратність  
✅ **Оптимізація пам'яті** - використання типу `int` для економії пам'яті  
✅ **Правильне іменування** - змінні названі відповідно до рекомендацій  

## Виконані вимоги практичної роботи:

1. ✅ Написаний універсальний клас `MatrixSorter` з методом для розв'язання задачи та тестовий клас `Main` у теці `src`
   - Основний клас не містить інтерфейсу користувача
   - Містить статичний метод `sortMatrix()` з логікою виконання завдання
   
2. ✅ Метод `main()` класу `Main` не містить ніякої логіки завдання
   - Лише перевіряє працездатність основного класу
   - Містить три тестові випадки з матрицями різних розмірів (2x2, 3x3, 4x4)
   
3. ✅ README.MD містить детальний опис завдання
   - Пояснення алгоритму роботи
   - Приклади з матрицями
   - Описання реалізації
   
4. ✅ Оптимізація програми:
   - Обрано оптимальні типи даних (`int` замість `Integer`)
   - Змінні названі правильно (`flattened`, `index`, `result`)
   - Мінімальне використання пам'яті
   
5. ✅ Javadoc-коментарі для основного класу:
   - Пояснення що саме обчислюється
   - Описання вхідних даних
   - Описання результатів
   - Інформація про виключення

6. ✅ Завдання виконано з використанням класу `Arrays`
   - Метод `Arrays.sort()` для сортування
   - Метод `Arrays.fill()` у тестах
   - Метод `Arrays.toString()` для виведення

## Результати тестування:

Програма виводить три тестові випадки:
- **Тест 1**: Матриця 3×3 з невпорядкованими елементами
- **Тест 2**: Матриця 4×4 з елементами у зворотному порядку
- **Тест 3**: Матриця 2×2 з мішаними елементами

Кожен тест демонструє коректність сортування матриці в порядку зростання елементів зліва-направо, зверху-вниз.
